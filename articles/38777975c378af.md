## TypeScriptとは？

TypeScriptは、ざっくり言うと **JavaScriptに型（Type）を追加した言語** です。
正式には「静的型システム」を備えた言語と言えます。

## 型注釈と型推論

静的型システムの特徴は、変数や式が型を持っていることです。

TypeScriptでは、変数定義を以下のように書くことができます。

```typescript
const cat: string = "neko";
```

これは、変数 `cat` が `string` 型であることを明示しています。
このような記述を **型注釈（type annotation）** と呼びます。

また、以下のような書き方も正しいTypeScriptのコードとして認識されます。

```typescript
const cat = "neko";
```

この場合、右辺の `"neko"` という値から `cat` が `string` 型であることをTypeScriptが自動的に判断してくれます。
この仕組みを **型推論（type inference）** と言います。

## 静的型システムのメリット

### 1. 型安全性

以下のようなコードを例に考えてみます。

```typescript
function nadenade(count: number): string {
  return "gorogoro".repeat(count);
}

console.log(nadenade("neko")); // エラー！
```

`nadenade` 関数は引数に `number` 型を受け取るように設計されていますが、
`"neko"` という `string` 型を渡そうとしています。

TypeScriptはこのような誤りを**コンパイル時点でエラーとして検出**してくれます。
（テキストエディタによっては、コードを書いた時点でエラー表示してくれます）

JavaScriptではこれがエラーにならないため、意図しない引数を渡していることに気づきにくいという問題がありました。TypeScriptはこのような問題を未然に防いでくれます。

### 2. コードのドキュメント化

TypeScriptではソースコード上に型情報が記載されます。
先ほどのコードを例にすると：

```typescript
function nadenade(count: number): string {
  return "gorogoro".repeat(count);
}
```

「`nadenade` 関数は `number` 型の引数を受け取り、`string` 型を返す」ということが、
関数の詳細な処理を読まなくても一目でわかります。

これにより、コードを初めて読む人でも理解しやすくなり、大規模な開発の現場では特に役立ちます。
