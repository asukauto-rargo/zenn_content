## はじめに

プログラムを書いていると、「式（Expression）」と「文（Statement）」という言葉を目にすることがあります。なんとなく使い分けているけれど、実際の違いをうまく説明できない——そんな方に向けて、TypeScript のコードを使いながら整理してみます。

---

## 「式」と「文」の基本

プログラムの流れは**文**によって構成されます。そして**式**は、その文を構成する要素です。

```typescript
const cat = "neko";
```

この1行を分解すると：

| 要素             | 内容                       |
| ---------------- | -------------------------- |
| 文（Statement）  | `const cat = "neko";` 全体 |
| 式（Expression） | `"neko"`                   |

式同士は演算子でつなぐこともできます。

```typescript
const cat = "neko" + "neko"; // => "nekoneko"
```

---

## 「式」と「文」の違い：結果の有無

2つの最大の違いは「**評価結果があるかどうか**」です。

- **式**：評価すると値（結果）が得られる
- **文**：直接的な結果を持たない

`"neko"` という式はそれ単体で `"neko"` という結果を持ち、`"neko" + "neko"` とつなぐと `"nekoneko"` という結果になります。一方、`const cat = "neko";` という文は、プログラムの状態を変えますが、それ自体が結果の値を返すわけではありません。

:::message
文は「宣言」と「その他」に分類されることもありますが、「式と文」の文脈では両者に本質的な違いはありません。
:::

---

## 「式文」とは？

ここで少し応用的な概念として、**式文（Expression Statement）** を紹介します。

> 式の後ろにセミコロン `;` を書いたもの

つまり、**式を文として扱う**記法です。

```typescript
console.log(nadenade());
```

この例を分解してみましょう：

- `nadenade()` は返り値を持つ関数 → **式**
- `console.log()` は返り値を持たない（`void`）→ **文**
- 全体として `;` で終わる → **式文**

式文は「式を実行したいが、その結果は使わない」というときに機能します。副作用（画面への出力、ログの記録など）だけを目的とした処理がこれにあたります。

---

## まとめ

| 概念                         | 定義                         | 例                            |
| ---------------------------- | ---------------------------- | ----------------------------- |
| 式（Expression）             | 評価すると値になるもの       | `"neko"`、`1 + 2`、`fn()`     |
| 文（Statement）              | プログラムの処理単位         | `const x = 1;`、`if (...) {}` |
| 式文（Expression Statement） | 式をセミコロンで終わらせた文 | `console.log("hi");`          |

「結果があるかどうか」を意識するだけで、コードの構造をより明確に捉えられるようになります。ぜひ日頃のコードを読む際の視点のひとつに加えてみてください。
