## はじめに

TypeScriptを学習する中で、`null`型や`undefined`型といった特殊な型の存在を知りました。
今回はそれらを含む**プリミティブ型**について、復習を兼ねて整理していきます。

## プリミティブ型とは？

プリミティブ型とは、TypeScript（JavaScript）における**最も基本的な値の型**です。
以下の7種類が存在します。

| 型          | 説明                         | リテラル例                          |
| ----------- | ---------------------------- | ----------------------------------- |
| `string`    | 文字列                       | `"hello"`, `'hello'`, `` `hello` `` |
| `number`    | 数値（整数・小数）           | `42`, `3.14`                        |
| `boolean`   | 真偽値                       | `true`, `false`                     |
| `bigint`    | 任意精度の整数               | `100n`                              |
| `null`      | 「値が存在しない」ことを明示 | `null`                              |
| `undefined` | 「値が未定義」であること     | `undefined`                         |
| `symbol`    | 一意な識別子                 | `Symbol("id")`                      |

:::message
プリミティブ型でない値はすべて**オブジェクト型**です。
オブジェクトはプリミティブを組み合わせて構成されます。
:::

## 各プリミティブ型の詳細

### 文字列型（`string`）

文字列を扱う型です。
プログラム上に直接記述された文字列は**文字列リテラル**と呼ばれます。

文字列リテラルには以下の書き方があります。

- **ダブルクォート**で囲む：`"hello"`
- **シングルクォート**で囲む：`'hello'`

また、**バッククォート**で囲む書き方もあり、これは**テンプレートリテラル**と呼ばれます。

```typescript
const name = "ねこ";
const greeting = `こんにちは、${name}！`;
console.log(greeting); // "こんにちは、ねこ！"
```

テンプレートリテラルには大きな特徴が2つあります。

1. **文字列中で改行ができる**
2. **`${式}` で式を埋め込める**

通常のクォートではどちらもできないため、複雑な文字列を組み立てる際に便利です。

---

### 数値型（`number`）

数値を扱う型です。整数と小数を区別しません。

```typescript
const nekoWeight: number = 5;
const pi: number = 3.14;
```

整数と小数を区別しないため、以下のような計算も自然に行えます。

```typescript
console.log(0.1 + 0.2); // 0.30000000000000004（浮動小数点の誤差）
console.log(0.5 + 0.5); // 1
```

:::message alert
`number`型は内部的にIEEE 754の64ビット浮動小数点数で表現されるため、`0.1 + 0.2` が正確に `0.3` にならないといった**浮動小数点の誤差**が生じることがあります。金額計算など精度が重要な場面では注意が必要です。
:::

プログラム上に数値を直接記述したものは**数値リテラル**と呼ばれます。

---

### BigInt型（`bigint`）

**任意精度の整数**を扱うプリミティブ型です（ES2020で導入）。

「任意精度」とは、どれだけ大きな整数でも誤差なく表現できるという意味です。
`number`型の整数精度は53ビット（約9千兆）までですが、`bigint`にはその制限がありません。

数値の末尾に `n` を付けた**BigIntリテラル**で記述します。

```typescript
const bigNekoWeight: bigint = 10n + 20n;
console.log(bigNekoWeight); // 30n
```

#### BigInt型の注意点

**整数のみ**を扱うため、割り算の結果が小数になる場合は切り捨てられます。

```typescript
const result: bigint = 5n / 2n;
console.log(result); // 2n（小数部分は切り捨て）
```

**`number`型との混合演算はエラー**になります。

```typescript
const result = 10n * 20; // ❌ コンパイルエラー
```

:::details numberとBigIntの使い分け
精度は`bigint`の方が高いですが、計算速度は`number`の方がはるかに高速です。
`number`の精度で十分な場合（ほとんどのケース）は`number`を使いましょう。

また、古いブラウザではサポートされておらず、Polyfill（古いブラウザで新しい機能を再現するスクリプト）での対応も難しいため、導入には環境の確認が必要です。
:::

---

### 真偽値型（`boolean`）

`true` か `false` の2値のみを扱うプリミティブ型です。

```typescript
const isHungry: boolean = true;
const isSleeping: boolean = false;
```

条件分岐（`if`文）やフラグ管理などで頻繁に使われます。

---

### null型とundefined型

どちらも**「データがない」ことを表す**プリミティブ型ですが、意味合いが異なります。

|          | `null`                         | `undefined`                      |
| -------- | ------------------------------ | -------------------------------- |
| 意味     | 「値が**意図的に**存在しない」 | 「値が**まだ定義されていない**」 |
| 代入     | 明示的に代入する               | 変数宣言時に自動で設定される     |
| `typeof` | `"object"`（歴史的な仕様）     | `"undefined"`                    |

```typescript
let a: string | undefined;
console.log(a); // undefined（初期化されていない）

let b: string | null = null;
console.log(b); // null（意図的に「空」を代入）
```

:::message
実務では、チームやプロジェクトによって「`null`を使う」「`undefined`に統一する」などの方針が決められていることが多いです。一貫性を保つことが大切です。
:::

### シンボル型（`symbol`）

:::details 💭 心の声：シンボル型って何？
シンボル型、聞き慣れないですよね。実際に使うケースは限られますが、ライブラリ開発などで重要な役割を果たします。
:::

シンボル型は、**他と絶対に被らない一意な値**を生成する型です。

```typescript
const neko = Symbol("id");
const nuko = Symbol("id");

console.log(neko === nuko); // false（同じ説明文でも別の値）
```

同じ `"id"` という説明を渡しても、生成されるシンボルは毎回異なります。

#### シンボルは何の役に立つ？

オブジェクトのプロパティ名に文字列を使うと、他の処理と名前が衝突する可能性があります。

```typescript
// 文字列キーだと衝突のリスクがある
const user = {
  id: 1,
};
```

シンボルを使えば、同じ `"id"` という説明を持っていても衝突しません。

```typescript
const id = Symbol("id");

const user = {
  [id]: 1,
};

console.log(user[id]); // 1
```

この性質により、以下のようなメリットがあります。

- ライブラリ内部でのみ使いたいプロパティ名を**安全に確保**できる
- 他のコードとプロパティ名が**衝突しにくい**
- 通常の文字列キーと**明確に区別**できる

#### シンボルの注意点

シンボルは文字列と暗黙的に連結できません。

```typescript
const s = Symbol("test");
console.log("value: " + s); // ❌ TypeError
```

文字列として扱いたい場合は、明示的に変換します。

```typescript
console.log("value: " + String(s)); // "value: Symbol(test)"
```

:::details 💭 心の声：シンボルの使いどころ
なるほど、概念はわかったけれど、メリットを実感するには実際のライブラリ開発やプラグイン設計で使ってみるのが良さそうですね。
:::

## まとめ

TypeScriptのプリミティブ型は7種類あり、それぞれ異なる役割を持っています。

普段よく使う `string`・`number`・`boolean` に加えて、`null` と `undefined` の使い分け、`bigint` の存在、そして `symbol` の一意性という特徴を押さえておくと、型への理解がぐっと深まります。
